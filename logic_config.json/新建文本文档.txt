# Role
你是一个资深的架构师和内容逻辑专家，擅长将文学创作逻辑转化为程序化的配置或大纲生成模型。

# Input: 小说底层逻辑拆解报告
- 作品流派: [例如：玄幻/系统流]
- 成功公式: [例如：弱势开局 + 极速打脸 + 宏大背景留白]
- 节奏模型: [例如：每3章一个冲突循环，每10章一个大高潮]
- 关键钩子(Hooks): [1.身份之谜; 2.三日之约; 3.神秘残片]

# Task
请基于上述逻辑，在 OpenCode 中为我完成以下任务（请根据需求三选一）：

## 方案 A：生成数据结构 (用于后端开发)
请将上述逻辑转化为一个 JSON 格式的配置文件，要求包含章节结构(chapters)、情绪值曲线(emotion_score)和伏笔索引(foreshadowing)。

## 方案 B：编写逻辑验证脚本 (用于自动化测试)
请用 Python 写一个简单的逻辑检查器，输入一段剧情大纲，判断它是否符合“成功公式”中的“弱势开局”和“极速打脸”这两个核心检测点。

## 方案 C：生成 Prompt 生成器代码 (用于写作辅助工具)
请编写一个函数，根据输入的“成功公式”，自动生成一段发给 LLM 的指令，要求其基于此公式扩充 5000 字的细纲。

# Output Requirement
请直接给出代码实现或结构化文档，并说明如何将拆解出的参数填入。
